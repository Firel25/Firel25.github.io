Hexo搭建

发表于 2018-03-22

Hexo搭建

文章旨在记录一下Hexo搭建过程的一个问题，以及解决办法。

整个Hexo的搭建过程按照这篇博客的基础篇和进阶篇的介绍进行搭建的，中途没有遇到什么大的问题，顺利搭建成功了。主要问题是在解决编辑博客内容插件Hexo-admin的安装的时候出了问题。

这个插件的安装只需要执行npm install –save hexo-admin即可。但是在配置该插件的时候需要设置登录密码，该密码采用了bcrypt的hash算法来避免明文密码存储，但是需要手动输入这个hash，有文章说可以通过各种语言C/Java/Python来进行计算该hash密文，于是选择了自己熟悉的Python，网上搜了一下bcrypt的简单用法计算自己密码的hash。

>>> import bcrypt
>>> print bcrypt.hashpw('test',bcrypt.gensalt())
$2b$12$SuSkzNcMRyr6AI6KXHcRx.xurgLz8UNJb3clLotMuivbz73Nq/gvy

将该执行结果，复制到配置文件中，开启服务，访问admin管理界面，出现了如下错误

Invalid salt revsion

看着是盐的生成出了问题，但是网上搜索了使用用法没有问题，于是去看了一下服务端的处理代码，在 /node_modules/hexo-admin/auth 目录下面，就两个文件，简单看了一下，验证代码如下：

if (request.body.username == config.username &&
        bcrypt.compareSync(request.body.password, config.password_hash)) {
        executionScope.success({name:request.body.user}, callback)
    }

只是简单的调用了compareSync函数，但是Python的包里面没有这个函数，只有checkpw函数，但是作用大致是一致的，就是检验明文执行相同算法与密文比较是否一致，难道是封装的不一样。然后想着尝试用node.js来实现总不会存在这样的问题了吧。于是搜索了一下node的bcrypt的用法，进行了如下测试：

> var bcrypt = require('bcryptjs');
undefined
> var salt = bcrypt.genSaltSync();
undefined
> var hash = bcrypt.hashSync("test", salt);
undefined
> bcrypt.hashSync("test", salt);
'$2a$10$y.BvuGqoax7tEWjFEEkyR.C1Sh6UUXEu2BiAIhLmXueVg0QQLQAPq'
> bcrypt.compareSync('test',hash)
true

然后重新配置密文，果然就可以了，问题也算解决了。但是我重新搜索了一下之前返回的错误，在stack-overflow一篇文章里面看到了下面的答复：

In particular: the salt JBCrypt expects salt version 2a, so a salt starting with $2a$. If you used another BCrypt version that uses a more modern salt version 2y or 2b - JBCrypt throw an exception.

是盐的版本出了问题，然后看了一下Python中gensalt的实现：

>>> help(bcrypt.gensalt)
Help on function gensalt in module bcrypt:
gensalt(rounds=12, prefix='2b')
>>>

可以看到Python中的默认版本好是2b，然后指定版本号重新尝试了一遍

>>> print bcrypt.hashpw('test',bcrypt.gensalt(12,'2a'))
$2a$12$4BESoo4y6DtHMUp4BUFbYeLdPLLMCsm9FYqTHZeXe3FmQBzHUQ/P6
>>>

配置好之后，尝试登录果然可以了。
